## Phase 1 — Component (1) pruning

- [DONE] 1.1 Stage A0 — design-pattern-aligned task decomposition. `surface/decompose.py` walks a C source tree, clusters .c files by subdirectory (e.g. `ipset/`, `ipvs/`) then by filename-prefix family (e.g. `nft_*`, `nf_conntrack_*`), pattern-labels each file via regex (ops_vtable, container_of, refcount, rcu, allocator, parser_sm), extracts function definitions via universal-ctags and per-cluster callee edges (intra + inter-cluster), and emits per-cluster JSON + `_index.json` summary. Run over the Phase-0.4 Linux 6.1.72 `net/netfilter/` tree: **245 sources → 24 clusters, 124,323 LOC**, dependency graph populated (e.g. `nft → {nf_tables, nfnetlink, xt, ...}`, `nf_tables → {nft_chain, core, nfnetlink, nft_flow, nft}`). Spot-check: CVE-2024-1086 sites land in sensible clusters — `nft_immediate.c` in `nft`, `nf_tables_core.c` in `nf_tables` (dominant patterns `allocator/rcu/parser_sm`), exactly the surface Phase 1.3 Stage-B contracts will sit on. Output: `surface/tasks/linux-6.1.72-netfilter/{24×cluster.json, _index.json}`. No LLM (Phase-1 rule); LLM-assisted relabeling is a Phase-3 hook. Soundness note: clustering quality affects how small/clean tasks are, not soundness — any partition stays sound so long as boundary contracts are verified in Stage B.
